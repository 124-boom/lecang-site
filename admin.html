使用说明

① 在左侧菜单选择要编辑的模块，填写或修改内容后点击右上角「保存所有更改」

② 保存后数据会自动同步到前台页面 index.html，刷新前台即可看到更新

③ 点击「恢复默认」可将所有内容重置为初始状态

④ 点击「导出配置」可下载当前所有内容的 JSON 文件，方便备份

⑤ 产品模块支持新增和删除，最多展示 8 个产品

⑥ 产品和禅意空间均支持上传图片（不超过 2MB），不上传则显示默认样式

⑦ 货币符号可在「品牌信息」中自定义修改

⑧ 每次保存会自动记录提交时间和内容，在「提交记录」中可查看并导出 Excel 表格

⑨ 客户在前台提交咨询后，在「客户信息」中可查看、导出 Excel（含姓名/电话/微信/意向/留言/时间）

⑩ 点击「📦 数据同步」配置云端代理，实现多设备实时同步（安全，网页不含密钥）

⑪ 配置云端后，点击「上传到云端」/「从云端下载」即可在多台设备间同步数据

⑫ 也可通过「导出全部数据」下载 JSON 文件，在其他设备手动导入（无需云端配置）

⚙️ 云端设置 未配置 ▼ 展开

本机服务器（双击「启动网站.bat」后在线）是你的数据源；网页不含任何服务器密钥。
云端可选两种：① 内网穿透服务器（家里常开电脑的花生壳/内网穿透地址）；② GitHub 仓库（把数据存在 GitHub，公开/私有均可，可自由增删）。
在每台电脑用同一个文件夹，选好云端类型、填好配置，点「双向同步」即可随时互通。

说明：花生壳免费版添加映射没有密码这一项，所以「隧道访问账号 / 密码」通常直接留空即可。 之前同步提示“管理密码错误”，其实是我们网站后台自己的「管理密码」字段（默认免密，留空就能同步），并不是花生壳要密码。 想给服务器加一道防护，可在家里的服务器用环境变量 ADMIN_KEY=你的密码 启动，那时才需要在这里填对应密码。

Token 在 GitHub → Settings → Developer settings → Personal access tokens 生成，勾选 repo 权限。 数据会通过 GitHub API 直接写入仓库（可自由增删），同一仓库的数据对全网公开可见，敏感客户信息请改用私有仓库。

勾选后，在花生壳后台保存会自动把内容推送到 GitHub 仓库，两个公网（花生壳 + GitHub Pages）同步更新； 在 GitHub Pages 后台保存则浏览器端直接推送。Token 存在本机服务/浏览器，不会随页面公开。
